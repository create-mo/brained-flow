Layer 2 — Sync (brain/ ↔ claude.ai)
→
wiki-watch.ps1 (background, auto-start at login)
Watches brain/projects/**/*.md for changes. On change, waits 5s debounce, then calls wiki-push.py for that project.
Auto · PowerShell FileSystemWatcher
↑
wiki-push.py — brain/ → claude.ai
Reads *.md from configured brain_path, compares with existing docs in the Project via API, deletes stale versions and uploads new ones.
HTTPS · claude.ai API session key auth
↓
wiki-pull.py — claude.ai → brain/ (on login)
Downloads all docs from claude.ai Project to local brain_path. Skips unchanged files. Triggered by WikiPull-OnLogin scheduled task.
At login · Windows Task Scheduler
⚙
wiki-sync-setup.ps1 — one-time setup
Registers WikiPull-OnLogin and WikiWatch-OnLogin in Windows Task Scheduler. Run once after installation.
Run once

⚠ Session key required: F12 → Application → Cookies → claude.ai → sessionKey
Save to ~/.claude/claude-ai-session.key · Expires after a few weeks · Never commit to git

Layer 3 — Skills (Cowork) & Commands (Claude Code / VS Code)
Cowork skills
brain-sync
Use the sync system — push, pull, watch, troubleshoot auth errors.
Auto-triggered by context
wiki-setup
Install and configure brain/ sync from scratch on a new machine.
Auto-triggered by context
brain-plan
Plan a task grounded in wiki context. Loads only relevant files (~62% context saved). Creates plans/*.md.
Auto-triggered by context
brain-run
Execute the active plan via subagents. Injects known-issues context per step. Writes solutions back to wiki.
Auto-triggered by context
Claude Code / VS Code commands
/brain-plan
Same as brain-plan skill. Invoked via slash command in terminal or VS Code. Creates plans/<date>_<slug>.md.
.claude/commands/brain-plan.md
/brain-run
Execute plan via subagents. Mark steps [x], [~], [!]. Save solutions to wiki. Suggest /brain-plan for blockers.
.claude/commands/brain-run.md
Install commands
Copy-Item commands\*.md .\.claude\commands\
or use install.ps1 — it asks for the target path.

Layer 4 — Full working cycle
1
Edit brain/ locally
Use Obsidian, VS Code, or any editor. [[wikilinks]] and YAML frontmatter are preserved.
Local Obsidian · VS Code
2
wiki-watch auto-pushes (5s debounce)
Changes appear in claude.ai Project. Mobile Claude sees current context immediately.
Automatic
3
Plan: brain-plan (Cowork) or /brain-plan (Claude Code)
Extracts task topic → calls /wiki <topic> to load only relevant wiki files → checks known-issues.md for existing blockers → spawns Plan agent with grounded context → writes plans/YYYY-MM-DD_slug.md.
Cowork Claude Code
4
Execute: brain-run (Cowork) or /brain-run (Claude Code)
Finds active plan → shows execution graph → waits for user confirmation → spawns subagents (parallel where possible) → verifies with TS/lint diagnostics → marks steps [x] done, [~] partial, [!] blocked.
Cowork Claude Code
5
Capture knowledge back to wiki
Subagents report findings → solutions.md gets non-trivial problems & fixes → known-issues.md is updated (distilled index) → .raw_nuances.md gets architectural invariants. Blocked steps become new brain-plan tasks.
Wiki files
6
wiki-watch auto-pushes updated wiki
Solutions and new knowledge land in claude.ai Project. Next session starts with full context — no re-explaining needed.
Automatic claude.ai
Installation
1
Clone the repo
git clone https://github.com/yourusername/brained-flow.git
2
Run install.ps1
Interactive. Asks for:
• Cowork skills directory (default: %APPDATA%\Claude\skills)
• Claude Code commands directory (default: .\.claude\commands\)
• brain/ wiki path (optional — for sync setup)
3
Set up wiki sync
Ask Claude in Cowork: "Help me set up wiki sync" — the wiki-setup skill guides you through session key, ORG_ID, scripts, and autostart.
4
Start planning
Say: "brain-plan: <your task>" in Cowork, or /brain-plan <task> in Claude Code.
